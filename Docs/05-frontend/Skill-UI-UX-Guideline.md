# UI/UX GUIDELINE — COWFARM SAAS

## Purpose
Mengatur standar pengalaman pengguna (UX) dan tampilan (UI) agar sistem mudah digunakan, konsisten, dan tidak membingungkan.

---

## Core Principle
UI/UX harus:
- Sederhana
- Jelas
- Cepat dipahami
- Tidak membebani user

---

## UX PRINCIPLES

### 1. Clarity First
User harus langsung paham fungsi halaman tanpa berpikir lama.

### 2. Minimal Cognitive Load
Jangan membuat user berpikir terlalu banyak untuk satu aksi.

### 3. Consistency
Semua halaman harus memiliki pola yang sama.

### 4. Feedback System
Setiap aksi user harus ada respon:
- sukses
- error
- loading

---

## UI STRUCTURE

### Dashboard Layout
- Sidebar (navigation)
- Topbar (user info)
- Content area (main data)

---

## DESIGN RULES

### Colors
- Primary: biru / hijau (trust & agriculture)
- Success: hijau
- Warning: kuning
- Danger: merah

---

### Typography
- Gunakan font clean (Inter / system font)
- Jangan terlalu banyak variasi font

---

### Spacing
- Gunakan spacing konsisten (4, 8, 16, 24, 32)
- Jangan terlalu rapat

---

## COMPONENT RULES

### Buttons
- Primary button untuk aksi utama
- Secondary untuk aksi tambahan
- Danger untuk delete

---

### Forms
- Label selalu di atas input
- Error message jelas di bawah input
- Input wajib punya placeholder

---

### Tables
- Gunakan pagination
- Gunakan search bar
- Gunakan action column (edit/delete)

---

## USER FLOW RULES

### Good Flow
Dashboard → List Data → Detail → Edit

### Bad Flow
Langsung banyak input tanpa konteks

---

## RESPONSIVE RULES
- Mobile first design
- Sidebar collapse di mobile
- Table scroll horizontal jika perlu

---

## UX ANTI PATTERNS
- Terlalu banyak tombol dalam satu halaman ❌
- Tidak ada feedback aksi ❌
- Navigasi membingungkan ❌
- Form terlalu panjang tanpa grouping ❌

---

## ACCESSIBILITY RULES
- Kontras warna jelas
- Font mudah dibaca
- Button jelas bisa diklik

---

## KEY PRINCIPLE
> "User should not think, system should guide"
