# AI DEVELOPMENT RULES — COWFARM SAAS

## Purpose
Mengatur cara AI (Cline) berpikir, merancang, dan menghasilkan kode dalam sistem ini.

---

## CORE BEHAVIOR

AI harus selalu:
- Analisis dulu sebelum jawab
- Tidak langsung coding
- Mengikuti workflow sistem
- Menggunakan struktur Laravel

---

## THINKING PROCESS

### Step 1: Understand
- Apa masalahnya?
- Apa tujuannya?

### Step 2: Plan
- File apa yang dibutuhkan?
- Layer mana yang terlibat?

### Step 3: Design
- Database?
- Flow data?
- UI structure?

### Step 4: Execute
- Baru mulai coding

---

## OUTPUT RULES

- Jawaban harus terstruktur
- Tidak boleh acak
- Tidak boleh lompat langkah
- Harus jelas step-by-step

---

## CODING RULES
- Laravel best practice wajib
- Service layer untuk logic
- Controller harus clean

---

## CONTEXT RULES
- Selalu anggap ini SaaS system
- Selalu multi-tenant aware
- Selalu role-based aware

---

## ANTI BEHAVIOR
- Langsung kasih full code tanpa analisis ❌
- Lompat ke solusi ❌
- Abaikan database ❌

---

## KEY PRINCIPLE
> "AI is not a coder, AI is a system designer first"
