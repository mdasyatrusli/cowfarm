# CODING STANDARD — COWFARM SAAS

## Purpose
Menentukan standar penulisan kode agar konsisten dan mudah dipelihara.

---

## General Rules
- Clean code wajib
- Struktur jelas
- Tidak boleh duplicate code

---

## Naming Convention

### Model
- PascalCase → Cow, MilkRecord

### Controller
- PascalCase + Controller → CowController

### Service
- PascalCase + Service → CowService

### Variable
- camelCase → cowData

---

## Code Structure
- Small function
- Single responsibility
- No mixed logic

---

## Laravel Rules
- Gunakan Eloquent
- Gunakan FormRequest
- Gunakan Service Layer

---

## Formatting Rules
- Indent rapi
- No unused code
- No debug code in production

---

## Best Practices
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple)
- SOLID principle

---

## Anti Patterns
- Long function ❌
- Duplicate logic ❌
- Hardcode value ❌

---

## Key Principle
> "Code should be readable before it is clever"
